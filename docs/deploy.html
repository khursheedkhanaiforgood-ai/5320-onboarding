Deployment Guide

How to get the 5320 Onboarding Agent running on a fresh MacBook — from zero to connected in under 10 minutes.

0 What You Need Before You Start

ItemNotes
MacBook (Apple Silicon or Intel)macOS Ventura 13+ recommended; older versions work
Python 3.11 or newerCheck: python3 --version in Terminal
Anthropic API keyGet one at console.anthropic.com → API Keys
USB-C serial console cableFTDI chip or Prolific PL2303 — both work on macOS
Extreme Networks 5320 switchPowered on, RJ45 console port accessible
Internet connectionRequired for Claude API calls and XIQ ZTP+

No Python 3.11? Install it from python.org/downloads or via Homebrew: brew install python@3.11

1 Install Homebrew (if not already installed)

Homebrew is the macOS package manager. Skip this step if you already have it (brew --version returns a version number).

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Follow the on-screen prompts. When it finishes, close and reopen Terminal.

2 Clone the Repository

Open Terminal and run:

git clone https://github.com/khursheedkhanaiforgood-ai/5320-onboarding.git
cd 5320-onboarding

This downloads all agent code into a folder called 5320-onboarding and moves you into it.

3 Create a Python Virtual Environment

A virtual environment keeps this project's dependencies isolated from the rest of your Mac.

python3 -m venv .venv
source .venv/bin/activate

You'll see (.venv) appear at the start of your Terminal prompt — that confirms it's active.

Every time you open a new Terminal window to run the agent, you must activate the environment again with source .venv/bin/activate from inside the 5320-onboarding folder.

4 Install Dependencies

pip install -r requirements.txt

This installs everything the agent needs: the Anthropic SDK, Rich terminal UI library, pyserial, and python-dotenv. Takes about 30 seconds on a fast connection.

5 Configure Your API Key

cp .env.example .env

Open the .env file in any text editor (TextEdit, VS Code, nano — your choice) and fill in your Anthropic API key:

ANTHROPIC_API_KEY=sk-ant-api03-...your-key-here...

Leave all other values at their defaults. The serial port is auto-detected.

Never commit .env to Git. It's already in .gitignore — just don't override that. Your API key is private.

6 Connect the Switch

1

Plug in the serial cable

USB-C end into your Mac. RJ45 end into the CON or CONSOLE port on the front of the 5320.

2

Verify macOS detects the cable

ls /dev/cu.usbserial-* /dev/cu.usbmodem* 2>/dev/null

You should see something like /dev/cu.usbserial-A9VKJO11. If nothing appears, try a different USB-C port or check the cable driver (FTDI or Prolific drivers are free downloads if needed).

3

Power on the switch

If it's already powered on, that's fine — the agent will pick up wherever the switch is in its boot sequence.

7 Run the Agent

python3 -m agent.main

The agent will:

  1. Auto-detect the USB-serial port (prompts you to choose if multiple cables are plugged in)
  2. Open a new Terminal window running screen connected to the switch console
  3. Start monitoring the console and displaying instructions in the original window

Two Terminal windows will be open:

  • Agent window (original) — shows status, current state, and instructions
  • Console window (new) — shows raw switch output; you type commands here

Optional flags:

python3 -m agent.main --port /dev/cu.usbserial-A9VKJO11   # specify port manually
python3 -m agent.main --verbose                            # extra debug output

8 What Happens Next

The agent will detect which situation applies and guide you accordingly:

Switch StateWhat the Agent Does
Fresh / Fabric EngineGuides you step by step through factory reset → ZTP+ → EXOS migration
Already running EXOSAuto-logs in, captures full config, saves a tabbed HTML report, opens it in your browser

For the fresh onboarding path, just follow the instructions shown in the agent window. The agent tells you exactly what to type and why at each step.

9 Stopping the Agent

Press Ctrl-C in the agent window at any time. The agent automatically:

  • Sends screen quit to close the console Terminal window
  • Releases the serial port so no manual cleanup is needed

After the already-onboarded flow completes, you'll also be offered:

  • Option 1 — Exit: closes everything cleanly
  • Option 2 — Direct config mode: keeps the screen session open for manual commands

11 Troubleshooting

SymptomFix
No serial ports foundCheck USB-C cable is plugged in; run ls /dev/cu.usbserial-* to confirm macOS sees it
ModuleNotFoundErrorVirtual environment not active — run source .venv/bin/activate first
AuthenticationError from AnthropicAPI key in .env is wrong or missing — double-check it at console.anthropic.com
Screen window doesn't openRun which screen — if empty, install with brew install screen
Agent exits immediatelyRun with --verbose to see the full error message
Port busy error on second runAgent cleanup handles this automatically; if it persists, run screen -ls and screen -X -S <id> quit
