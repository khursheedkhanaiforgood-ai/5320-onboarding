What Changes vs What Stays

Unchanged — Path A

SW1 + AP1

SW1 (192.168.0.28) remains EXOS/SwitchEngine. VLAN 40, DHCP, ipforwarding, Quantum Fiber static route all stay as-is. AP1 (192.168.0.12) stays on SW1. No changes to existing XIQ policy or config.

Changed — SW2

EXOS → VOSS/FabricEngine

SW2 (currently 192.168.0.11, EXOS, VLAN 70) will be wiped and re-imaged to VOSS/FabricEngine. New OS, new CLI paradigm, new Fabric Connect architecture. All existing VLAN 70 config will be replaced.

New Device — IPE

IPE (to be confirmed)

New device/role inserted between SW2 and the HomeModem. Handles the connectivity path to Extreme Networks RDC in Raleigh. Device type, model, software, and connectivity protocol TBD — see Open Questions.

Changed — AP2

AP2 re-homed to VOSS SW2

AP2 (192.168.0.25) will be re-adopted under the new VOSS/Fabric architecture. New XIQ policy, new SSIDs, new VLAN scheme TBD. AP2 connects to SW2 post-upgrade.

New — RDC Services

Extreme Networks RDC, Raleigh

Path B terminates at Extreme Networks Regional Data Centre. Service offerings accessible via this path TBD — could include XIQ private cloud, XMC, Fabric services, EVPN, SD-WAN, or lab/demo infrastructure.

Unchanged — HomeModem

Shared WAN uplink

HomeModem (192.168.0.1) is the shared internet gateway for both paths. No changes needed to the modem itself. Quantum Fiber static routes for Path A (10.40.0.0/24) remain. New routes for VOSS/Fabric subnets TBD.

Open Questions — Must Resolve Before Coding

These must be answered before any configuration can be written. They define the architecture, IP scheme, protocols, and device procurement.
  • Q1
    What exactly is the IPE?
    Device model? Physical appliance or virtual? Is it an Extreme Networks product (e.g. VSP running VOSS as a fabric edge, or a dedicated WAN edge device)? Does "IPE" stand for a specific Extreme term — IP Fabric Edge, Intelligent Policy Engine, something else? This is the most critical unknown — everything else depends on it.
  • Q2
    How does the IPE connect to the RDC?
    What transport/protocol: IPsec VPN tunnel? Fabric Extend (VXLAN over IP)? SPB (Shortest Path Bridging) over WAN? GRE? SD-WAN overlay? The protocol determines the VOSS Fabric architecture and the configuration required on both ends.
  • Q3
    What services are available at the Extreme Networks RDC Raleigh?
    XIQ private cloud? XMC (Extreme Management Center)? Fabric Connect demo lab? EVPN/VXLAN services? PoC infrastructure? This defines what we're integrating toward and shapes the end-to-end test plan.
  • Q4
    VOSS upgrade path for SW2?
    How does SW2 go from EXOS SwitchEngine 33.x to VOSS/FabricEngine? Is there a conversion tool or is it a clean flash? What VOSS version targets? Does SW2 (5320-16P-2MXT-2X) support VOSS natively or does it need a specific firmware?
  • Q5
    Fabric architecture scope?
    Will SW2 run a full SPB Fabric Connect domain? Or is this a simpler VOSS deployment without Fabric (just VOSS CLI + SVIs)? If Fabric Connect, what are the B-VLANs, IS-IS instance, and I-SID assignments?
  • Q6
    New IP addressing scheme for Path B?
    VLAN 70 (10.70.0.0/24) was the old scheme on SW2 under EXOS. What replaces it in the VOSS world? New subnets for client VLANs on AP2? New management IP for SW2?
  • Q7
    XIQ policy for VOSS SW2?
    Will SW2 (VOSS) still be managed via XIQ? Or does VOSS use XMC instead? If XIQ, does it support VOSS/FabricEngine switch templates? What's the onboarding flow — ZTP+ still valid for VOSS?

Phase 1 — SW2 VOSS Upgrade (detail)

  • 1
    Confirm VOSS image availability for 5320-16P-2MXT-2X
    Verify the 5320 hardware supports VOSS/FabricEngine. Download correct firmware from Extreme Networks portal. Note: 5320 ships with EXOS — confirm conversion procedure (conversion image vs clean flash).
  • 2
    Delete SW2 from XIQ before wipe
    XIQ → Manage → Devices → Delete SW2. Otherwise ZTP+ will re-push EXOS config on next boot.
  • 3
    Wipe SW2
    Connect via console (/dev/cu.usbserial-A9VKJO11, 115200 baud). Run: unconfigure switch all → confirm y. Then flash VOSS image per conversion procedure.
  • 4
    VOSS initial config
    Set hostname, management IP, SSH credentials. Verify VOSS CLI access. Key difference: VOSS uses enable → config terminal paradigm vs EXOS. Default credentials on VOSS: rwa/rwa.
  • 5
    Clear SSH known_hosts on Mac after wipe
    ssh-keygen -R 192.168.0.11 before reconnecting.
