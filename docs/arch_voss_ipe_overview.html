Two-tier edge: SW2 (VOSS) runs the IAH environment internally and connects via a physical uplink port to the IPE — a separate physical box. The IPE sits between SW2 and the HomeModem, handling the WAN-facing side of the Fabric Extend tunnel to the RDC. SW2 handles the fabric/I-SID logic; the IPE handles the internet path.

IPE — Physical Box (Integrated Application Hosting Edge)

Confirmed Apr 30 2026: The IPE is a physical device — not a virtual service inside SW2. It is a dedicated Extreme Networks appliance (Integrated Application Hosting Edge) that physically sits between SW2 and the HomeModem. Device model and specs to be confirmed.
Physical Placement

SW2 uplink → IPE → HomeModem

IPE is cabled between SW2's uplink/IAH port and a LAN port on the HomeModem. It is the physical WAN gateway for Path B. All traffic from SW2 destined for the internet or RDC exits through the IPE.

Fabric Role

Fabric Extend Tunnel Endpoint

The IPE is the local endpoint of the Fabric Extend tunnel. It encapsulates SW2's I-SIDs into IP/IPsec frames for transport across the HomeModem/internet to the Extreme Networks RDC in Raleigh — the far-end FE endpoint.

Application Hosting

Edge Compute Platform

The IPE hosts applications at the network edge (Docker-based, similar to VOSS IAH). Runs service connectors, analytics agents, or RDC-managed applications locally — reducing latency vs pure cloud execution.

Traffic Flow — Step by Step

① iPhone → AP2 (Wi-Fi)
② AP2 → SW2 port (Fabric Attach — AP2 signals I-SID via LLDP/FA TLVs)
③ SW2 auto-sense maps AP2 service to I-SID → no manual port config needed
④ SW2 encapsulates traffic in SPB / I-SID frame
⑤ SW2 IAH virtual interface → physical uplink port → IPE (physical box)
⑥ IPE: Fabric Extend encapsulation (wraps I-SID in IP + IPsec header)
⑦ IPE → HomeModem → Internet  (Fabric Extend tunnel)
⑧ Internet → Extreme Networks RDC, Raleigh NC (FE remote endpoint)
⑨ RDC decapsulates → delivers to service offering

Fabric Architecture — VOSS / SPB on SW2

IS-IS Control Plane

SW2 Fabric Identity

First task after VOSS upgrade: establish IS-IS Control Plane. SW2 gets Nickname 0.00.02 — its unique identity in the Fabric domain. IS-IS discovers topology and builds the SPB forwarding tree across the Fabric Extend tunnel to the RDC.

I-SIDs

Service Identifiers (replace VLANs)

I-SIDs are the service abstraction in Fabric Connect. Each wireless service (SSID) on AP2 maps to an I-SID on SW2. Fabric Extend wraps these I-SIDs in IP headers so they traverse the internet to the RDC as if on a local cable.

Fabric Extend (FE)

WAN Tunnel — SPB over IPsec

FE-Tunnel configured on SW2: source = SW2/IPE WAN IP, destination = RDC gateway Raleigh. Carries I-SIDs transparently across the HomeModem. No VPN client needed — FE is native VOSS. IS-IS adjacency forms across the tunnel.

Fabric Attach (FA)

AP2 Zero-Touch Provisioning

fa enable + auto-sense enable on the SW2 port connected to AP2. AP3000 sends FA LLDP TLVs requesting its I-SIDs. SW2 auto-provisions — no manual VLAN or I-SID tagging on the port. True zero-touch AP onboarding.

Phase 1 — SW2 VOSS Upgrade (Pre-Flight)

Prerequisite: Delete SW2 and AP2 from XIQ before rebooting — otherwise ZTP+ re-pushes EXOS config on first boot.
  • 1
    Delete SW2 + AP2 from XIQ
    XIQ → Manage → Devices → select SW2 (192.168.0.11) + AP2 (192.168.0.25) → Delete. Confirm.
  • 2
    Console cable to SW2
    screen /dev/cu.usbserial-A9VKJO11 115200
  • 3
    Reboot and catch bootloader
    boot from EXOS CLI (or power cycle). Press Space Bar immediately at the bootloader prompt. Timing is critical — if missed, power cycle and retry.
  • 4
    Select VOSS conversion
    In bootloader menu: select "Change the switch OS to Fabric Engine (VOSS)". Confirm. Switch downloads VOSS image, applies, reboots. All EXOS config is wiped.
  • 5
    First VOSS login
    Default: rwa / rwa. Run show sys-info to confirm VOSS version and hostname.
  • 6
    Execute VOSS Pre-Flight CLI
    Hostname, management IP, SSH keys, IS-IS, Nickname 0.00.02, B-VLANs. Script ready to generate on your go-ahead.
  • 7
    Clear Mac SSH known_hosts
    ssh-keygen -R 192.168.0.11 before SSH to new VOSS management IP.
