Phase 1 — SW1 Factory Reset
01 Factory Reset Command — Wrong Syntax Hiccup
Error

Typed delete configuration primary — got ^% Invalid input detected. This is Fabric Engine (EOS) syntax, not EXOS.

Lesson

EXOS factory reset command is unconfigure switch all. "Delete configuration primary" is Fabric Engine only. Don't mix the two.

* SW.5 # delete configuration primary
^ %% Invalid input
* SW.7 # unconfigure switch all
Restore all factory defaults and reboot? (y/N) Yes
Clearing SSH private key... The system is going down NOW!
02 Copy-Paste Hidden Characters Hiccup
Error

Multiple ^% Invalid input errors when pasting commands from the terminal or HTML. Hidden Unicode characters (zero-width spaces, smart quotes) were injected during copy.

Lesson

Always type EXOS commands manually — never paste. Even a single hidden character breaks the parser with a cryptic ^ marker.

Error

Wizard prompted for new admin account name. Typed admin → Error: User Account already exists. Even after factory reset, the built-in admin account persists.

Lesson

EXOS keeps the built-in admin account through factory reset. The enhanced security wizard creates an additional admin account — use a different name (e.g. babla).

Key Lesson (from Mar 27)

ipforwarding must be enabled on every routed VLAN — including Default. If Default VLAN has no ipforwarding, inter-VLAN traffic can't exit to the internet even if Corp1/Guest1 do. This was the root cause of the Mar 27 iPhone internet failure.

09 ARP Table vs Route Table — What's the Difference? Q&A / Learning

Question: "Why does show iproute show 192.168.0.24 but show iparp shows 192.168.0.28?"

Explanation

iproute is the switch's own routing table — it shows the switch's own IP addresses and known prefixes. The switch's IP (192.168.0.24) appears here as its directly connected Default VLAN address.

iparp shows other devices the switch has discovered via ARP. A device never ARPs for itself, so 192.168.0.24 never appears in the ARP table. The .28 entry is the AP3000 that just booted on Port 3 and received a DHCP lease from the home router.

show iparp
VR-Default 192.168.0.28 d8:e0:16:37:6c:00 8 NO Default 1 Port3
# ↑ This is the AP3000 — new DHCP lease after PoE came up on Port 3

13 AP Not Appearing in XIQ — Delete vs Reset & Delete Key Hiccup
Root Cause

AP3000 was removed from XIQ using "Delete" (not "Reset and Delete"). "Delete" removes the device from XIQ's database but leaves the AP's local flash with the old org ID and server binding. On reboot, the AP phones home to XIQ but is rejected — it's bound to a stale org entry that no longer exists in XIQ.

Lesson

Always use "Reset and Delete" in XIQ when decommissioning an AP. This sends a factory-reset command to the AP before removing it, clearing the org binding on the device itself. If the AP is already off-network, do a physical factory reset (hold pinhole 10 seconds while powered).

Fix applied: physical pinhole reset on AP3000-A. AP rebooted → new DHCP lease → 192.168.0.26. Firmware update triggered automatically (reached 54% and counting at time of writing).
