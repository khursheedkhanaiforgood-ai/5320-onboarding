Module A — Architecture Design
03 Does SW1 Need a Separate Router? Q&A
Q: Do I need a separate L3 device between the switch and Quantum Fiber?
No. The EN-5320 is a combined L2/L3 switch. With ipforwarding enabled and IP addresses on each VLAN, the switch performs inter-VLAN routing itself. The only external device needed is Quantum Fiber for internet NAT — and it needs static routes back to 10.20/10.30 subnets via 192.168.0.24.
04 SSH Stability — L2 Discovery & DHCP Renewal Q&A
Q: SW1 and SW2 are on the same L2 — my SSH session will drop when DHCP renews?
Yes — both switches start on the home router DHCP pool (192.168.0.0/24). If a lease renews and the IP changes, the SSH session drops. Reconnect using the new IP. This is exactly why VLAN 10 (static 10.10.0.1/10.10.0.2) was designed as the permanent inter-switch management path for Phase 3.

08 Why Does Port 3 Need Tagged on Both VLANs? Q&A
Q: Why is Port 3 tagged on Corp1 AND Guest1?
An AP with two SSIDs needs to carry both VLAN 20 and VLAN 30 over a single cable. 802.1Q tagged (trunk) mode allows this — the AP uses the VLAN tag to map each SSID to the correct VLAN. Default VLAN stays untagged on Port 3 so the AP can send XIQ registration traffic untagged during provisioning.

Corrections

EXOS creates a VLAN with name and tag in one command: create vlan Corp1 tag 20. No separate rename step. The VR-Default port manipulation is unnecessary — EXOS automatically assigns data ports to VR-Default. Only management port assignments require explicit VR config.

11 AP Factory Reset — XIQ Option vs Physical Q&A
Q: In XIQ I see "Reset to Template Default" but no "factory default" option?
"Reset to Template Default" reverts the AP's XIQ policy template — it does NOT do a factory reset. Since the APs were already deleted from XIQ, this option is irrelevant. A true factory reset requires the physical pinhole button on the AP: hold for 10 seconds while powered.
12 Terminal History Lost — zsh Fix Q&A
Q: My terminal history gets cleaned up after a while — how do I fix this? (Shell: zsh, Terminal: Terminal.app)
Add to ~/.zshrc:
HISTSIZE=100000
SAVEHIST=100000
setopt HIST_IGNORE_DUPS
setopt SHARE_HISTORY
Then run source ~/.zshrc.

16 Copy-Paste Hidden Characters Hiccup
Error

Multiple ^% Invalid input errors when pasting EXOS commands from the terminal or HTML. Hidden Unicode characters (zero-width spaces, smart quotes) injected during copy broke the EXOS parser.

Rule

Always type EXOS commands manually — never paste from a browser or rich-text terminal. Even one hidden character triggers a cryptic ^ marker error.

22 ARP Table vs Route Table Q&A
Q: Why does show iproute show 192.168.0.24 but show iparp shows 192.168.0.28? Why are they different?
iproute is the switch's own routing table — 192.168.0.24 is the switch's own IP on Default VLAN. iparp shows neighbor devices discovered via ARP. A device never ARPs for itself, so 192.168.0.24 never appears in the ARP table. The .28 entry (MAC d8:e0:16:37:6c:00, age 8 min) is the AP3000 that just powered up on Port 3 and pulled a DHCP lease from the home router.

26 AP Not Appearing in XIQ — Delete vs Reset & Delete Key Hiccup
Root Cause

APs were removed from XIQ before the session using "Delete". "Delete" removes the device record from XIQ but leaves the AP's local flash with its old org ID and server binding. On reboot the AP contacted XIQ and was rejected — it tried to register under a stale org entry that no longer existed.

Rule

Always use "Reset and Delete" in XIQ. This sends a factory-reset command to the AP before removing the record, clearing the org binding on the device itself.

If the AP is off-network (can't receive the remote reset): physical pinhole button, 10 seconds while powered.

Fix: physical pinhole reset on AP3000-A. New DHCP lease → 192.168.0.26. Automatic firmware update triggered (54% at context break, completed shortly after).

Phase 2 — Troubleshooting
30 Two "Default" in show iproute — What Does it Mean? Q&A
Q: Why do I have two "Default" in show iproute?
Default Route in the Destination column = the routing concept 0.0.0.0/0 — "send everything unknown to 192.168.0.1 (Quantum Fiber)". This is a routing term, not a VLAN name.

Default in the VLAN column = the VLAN named "Default" (VID 1) — SW2's management IP 192.168.0.28. Same VLAN, two completely different contexts in the same table.
31 Cannot Ping 8.8.8.8 from Corp2/Guest2 Gateways Troubleshooting
Symptom

Could not ping 8.8.8.8 sourced from 10.50.0.1 or 10.60.0.1. show vlan confirmed ipforwarding enabled (f flag) on all three VLANs. show iproute confirmed default route → 192.168.0.1 exists. Quantum Fiber static routes for 10.50/10.60 → 192.168.0.28 confirmed in place.

Root Cause — EXOS Ping Syntax

Connectivity was fine all along. The issue was EXOS ping argument order. ping 8.8.8.8 from 10.50.0.1 and ping vr VR-Default 8.8.8.8 from 10.50.0.1 with wrong order both fail. The from keyword must follow the destination.

Correct: ping vr VR-Default 8.8.8.8 from 10.50.0.1

ping vr VR-Default 8.8.8.8 from 10.50.0.1
4 packets, 0% loss, avg 5ms ✓
ping vr VR-Default 8.8.8.8 from 10.60.0.1
4 packets, 0% loss, avg 5ms ✓
