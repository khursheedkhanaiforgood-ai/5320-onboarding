Local Deployment — No Railway

Run the agent entirely on your Mac. No cloud account, no subscription, no server. You and the switch, that's it.

0 What You Need

ItemNotes
MacBook (Apple Silicon or Intel)macOS Ventura 13+ recommended
Python 3.11 or newerCheck: python3 --version
Anthropic API keyGet one at console.anthropic.com
USB-C serial console cableFTDI or Prolific PL2303
Extreme Networks 5320 switchPowered on, console port accessible

1 Clone the Repository

git clone https://github.com/khursheedkhanaiforgood-ai/5320-onboarding.git
cd 5320-onboarding

2 Create a Virtual Environment

python3 -m venv .venv
source .venv/bin/activate

Every new Terminal session: run source .venv/bin/activate before running the agent.

3 Install Dependencies

pip install -r requirements.txt

4 Set Your API Key

cp .env.example .env
# Open .env and set ANTHROPIC_API_KEY=sk-ant-...

5 Connect the Switch

USB-C end into Mac. RJ45 end into the CON / CONSOLE port on the 5320. Verify:

ls /dev/cu.usbserial-*

6 Run

~/5320-onboarding/run.sh

Or from inside the folder with the venv active:

python3 -m agent.main

Railway Deployment — Remote Visibility

The agent runs in the cloud. A field tech connects the bridge on their Mac. An engineer watches and guides from any browser, anywhere.

ⓘ Why Railway?

ScenarioLocalRailway
Solo operator, own Mac✓ PerfectOverkill
Manager wants to watch live✗ Can't✓ Anyone with URL
Remote hands — field tech onsite, engineer remote✗ Can't✓ Engineer sees console
Audit trail — central session log✗ Local only✓ Visible to all
No Anthropic API key needed by field tech✗ Each user needs one✓ One key on server

A One-Time Server Setup (Admin)

Done once by whoever owns the Railway account. Field techs never touch this.

1

Create Railway project

Go to railway.app → New Project → Deploy from GitHub → select khursheedkhanaiforgood-ai/5320-onboarding, branch feature/railway.

2

Set environment variables

In Railway → your service → Variables, add:

VariableValue
ANTHROPIC_API_KEYsk-ant-api03-... (your key)
BRIDGE_TOKENAny secret string you choose — share this with field techs
3

Generate a public URL

Railway → your service → Settings → Networking → Generate Domain. You get a URL like https://5320-agent.railway.app. Share this URL and the BRIDGE_TOKEN with your field techs.

B Field Tech — Run the Bridge

Done by the person physically connected to the switch. No Railway account needed, no API key needed.

1

Install bridge dependencies

pip install pyserial websockets
2

Get the bridge script

git clone https://github.com/khursheedkhanaiforgood-ai/5320-onboarding.git

Or download just bridge/bridge.py from GitHub directly.

3

Plug in the console cable

USB-C into Mac. RJ45 into the CON / CONSOLE port on the 5320.

4

Run the bridge

python3 bridge/bridge.py \
  --server wss://5320-agent.railway.app \
  --token YOUR_BRIDGE_TOKEN

The bridge auto-detects the serial port. If multiple cables are plugged in it will ask you to choose.

That's it for the field tech. No API key, no venv, no config file — just two pip packages and one command.

C Observer / Engineer — Open the Web UI

Anyone who needs to watch or guide the session — open the Railway URL in any browser:

https://5320-agent.railway.app

You will see:

  • Left pane — live switch console output, scrolling in real time
  • Right pane — agent instructions (what to type, why, current state)
  • Bottom bar — manual command input if you need to send something directly

The Bridge Connected indicator in the top right confirms the field tech's cable is live.
