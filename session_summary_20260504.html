► Part 1 — 4-Way Handshake Deep Dive
01 Key Hierarchy — WPA2-PSK vs WPA3-SAE
WPA2-PSK PATH — No Forward Secrecy
PASSWORD (Static)
└─ PBKDF2(password, SSID, 4096 iterations)
PMK — Pairwise Master Key (static, 256-bit)
└─ PRF(PMK, "Pairwise key expansion" ‖ AA ‖ SPA ‖ ANonce ‖ SNonce)
PTK — Pairwise Transient Key (per session)
KCK — Key Confirmation Key → MIC computation
KEK — Key Encryption Key → encrypts GTK in M3
TK — Temporal Key → encrypts data frames
GTK — Group Temporal Key → broadcast/multicast decrypt
WPA3-SAE PATH — Perfect Forward Secrecy
PASSWORD + DH Exchange (ephemeral)
└─ SAE Commit/Confirm → fresh DH keypair each session
PMK — Ephemeral (dies after session)
└─ Same PTK derivation as WPA2
PTK / KCK / KEK / TK → identical structure
⚠ WPA2-PSK VULNERABILITY
Attacker captures M1+M2 (ANonce+SNonce visible in cleartext). Tries passwords from dictionary → derives candidate PMK → computes candidate PTK → checks MIC in M2. Match = password found. ALL past sessions decryptable retroactively.
✓ WPA3-SAE PROTECTION
Even if attacker gets password later — cannot reconstruct ephemeral DH private keys → cannot re-derive PMK → past sessions protected forever. Forward secrecy = past traffic stays safe.

07 Concepts Mastered This Session
✓
PMK Origin Paths — WPA2-PSK: PBKDF2(password) = static PMK. WPA3-SAE: DH exchange = ephemeral PMK. No RADIUS = PSK path.
✓
PTK Derivation — PRF(PMK, label, AA, SPA, ANonce, SNonce) → KCK + KEK + TK. All inputs visible in cleartext except PMK.
✓
MIC Purpose — Integrity only, not encryption. M2 MIC proves client knows PMK. Computed with KCK (subset of PTK).
✓
KEK vs TK — KEK encrypts GTK inside M3 before TK is installed. Install bit = load TK into hardware crypto engine.
✓
GTK Impact — Group key decrypts ALL broadcast/multicast. Cracking GTK reveals ARP, DHCP, mDNS — full VLAN topology.
✓
Replay Counter — Prevents KRACK attack. MIC proves integrity; Replay Counter proves freshness. Both required.
✓
Forward Secrecy — WPA3-SAE: DH ephemeral keys discarded after session. Password leak cannot decrypt past captures.
✓
NAV / Virtual Carrier Sense — Duration field in every frame. RTS sets NAV, CTS decrements. CTS-to-Self Duration=0 resets all timers.
✓
RTS/CTS Directionality — Either side initiates. AP sends 70% of RTS in this capture (heavy downlink to 24 clients).
✓
Monitor Mode vs AP Capture — RSSI on AP TX frames = third-party sniffer. All frames show receive signal level.
✓
Rogue Device Fingerprinting — Unknown OUI + locally-administered MAC bit + single frame type + no association = NAV attack signature.
