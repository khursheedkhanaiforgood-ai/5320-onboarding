What Happened
SW2 was found with a DHCP-learned default route pointing directly to the modem (192.168.0.1) — left over from when SW2 Port 1 was cabled to the modem. That cable was physically disconnected, leaving SW2 with a stale route to an unreachable gateway.

Additionally, Default VLAN (VLAN 1) was not on Port 10 (the inter-switch trunk). This meant AP2 — which boots untagged on VLAN 1 and needs to reach XIQ — had no path from SW2 to SW1 to the internet. AP2 was offline.

With SW2 Port 1 now disconnected from the modem, the original loop triangle risk is eliminated. VLAN 1 on the Port 10 trunk is now safe and necessary.
✓ XIQ ONLINE — Both APs confirmed up
Root Cause Analysis
Two Issues

Issue 1 — Bad Default Route on SW2

Ori  Destination     Gateway      Flags        VLAN
#bo  Default Route   192.168.0.1  UG-D---um    Default   ← DHCP-learned, wrong gateway
DHCP-learned The #bo origin flag = BOOTP/DHCP. Cannot delete with configure iproute delete — returns error. Must disable DHCP client on the VLAN to remove it.

SW2 Port 1 was previously cabled to the modem. When disconnected, the DHCP lease expired but the route remained stale. Gateway 192.168.0.1 was unreachable — SW2 had no path to it.

Issue 2 — Default VLAN Missing from Port 10 Trunk

Default VLAN was intentionally removed from Port 10 during the Apr 21 build to prevent a loop triangle:

Modem → SW1 Port 1 → Default VLAN → Port 10 → SW2 → SW2 Port 1 → Modem

That loop required SW2 Port 1 to be connected to the modem. That path no longer exists. Without Default VLAN on Port 10, AP2 (on SW2 Port 3, untagged) had no way to forward its DHCP request to SW1 and out to XIQ.

Why the Loop Triangle is Gone

The loop required a second path from SW2 back to the modem. SW2 Port 1 was that path. Now that Port 1 is disconnected, VLAN 1 can safely travel on the Port 10 trunk — there is only one path: AP2 → SW2 Port 3 → VLAN 1 → Port 10 trunk → SW1 → Port 1 → Modem.
Fix Applied
Commands Run

SW2 — Step 1: Remove Bad Default Route

disable dhcp vlan Default
Kills the DHCP client on Default VLAN. Auto-removes the DHCP-learned default route. Routing table → 0 routes. SW2 is now pure L2.

SW2 — Step 2: Add VLAN 1 to Trunk

configure vlan Default add ports 10 tagged
save configuration
VLAN 1 now crosses the Port 10 trunk. AP2 untagged traffic on Port 3 → forwarded via Port 10 → reaches SW1 → internet → XIQ.

SW1 — Mirror the Trunk

configure vlan Default add ports 10 tagged
save configuration
SW1 Port 10 must also accept tagged VLAN 1 frames from SW2. Both ends of the trunk must agree.
