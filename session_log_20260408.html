configure vlan SW_VLAN_20 dhcp-options default-gateway 10.20.0.1
configure vlan SW_VLAN_20 dhcp-options dns-server primary 8.8.8.8
configure vlan SW_VLAN_20 dhcp-options dns-server secondary 8.8.4.4

Lessons

Key Lessons — April 7 + 8 2026

⚠ enable ipforwarding vlan Default — mandatory after every factory reset

Without this, clients get DHCP and a gateway but Safari times out. Return traffic from internet arrives on VLAN 1 and the switch won't forward it. Not pushed by XIQ.

⚠ DHCP syntax — use these exactly (verified on SwitchEngine 33.x)

dhcp-address-range 10.20.0.100 - 10.20.0.200 (dash with spaces)
dhcp-options default-gateway (not "gateway")
enable dhcp ports 3 vlan X (port number required)

⚠ VLAN SVI stays DOWN with 0 ports — add port 1 tagged to bring up

EXOS requires at least one port in link-up state for SVI to be active. Fix: configure vlan SW_VLAN_10_Mgmt add ports 1 tagged

⚠ Never add VLAN 1 to XIQ Routing section

Breaks XIQ ↔ switch communication immediately. VLAN 1 is the XIQ management lifeline — leave it unmanaged.
