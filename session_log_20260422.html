Session 1

L2 Fundamentals — MAC, FDB, VLANs, STP · Apr 20 · 10 Questions

Q1
What happens when a switch receives a frame for a MAC address it has never seen before?
The switch floods the frame out all ports except the one it arrived on. This is unknown unicast flooding. It then learns the source MAC and port for future forwarding.
Q2
What is the Forwarding Database (FDB) and what does each entry contain?
The FDB (MAC table) maps MAC address → port → VLAN → age timer. When a frame arrives, the switch records the source MAC, the ingress port, and the VLAN tag. Subsequent frames to that MAC are forwarded directly rather than flooded.
Q3
Difference between an access port and a trunk port?
Access port: carries traffic for one VLAN, untagged. Client devices attach here. Trunk port: carries multiple VLANs, 802.1Q tagged. Switch-to-switch or switch-to-AP links use trunks.
Q4
Why does STP (Spanning Tree Protocol) exist?
To prevent broadcast storms in networks with redundant switch links. Without STP, frames would loop forever. STP elects a root bridge and blocks redundant ports, creating a loop-free topology.
Q5–10
BPDU, root bridge election, port roles, VLAN segmentation benefits, ARP at L2/L3 boundary, inter-VLAN ping test setup
BPDUs are STP hello messages exchanged between switches to elect the root bridge (lowest bridge ID wins). VLANs segment broadcast domains — a frame in VLAN 10 cannot reach VLAN 20 without L3 routing. ARP resolves IP→MAC within a subnet. Inter-VLAN ping requires L3 (router or SVI) — a bridge cannot route between VLANs.
Session 2

L2 Applied — Inter-VLAN Bridge, LLDP, VLAN 1 Loop · Apr 22 · 8 Questions

Opening Q (posed Apr 21)
KK to explain why inter-VLAN ping between VLAN 10 and VLAN 20 requires more than a bridge. KK correctly identified: bridges operate at L2 and cannot route between VLANs — need an SVI or router.
Q1
You have SW1 and SW2 connected on Port 10. Both have clients in VLAN 10. A client on SW2 can't ping SW1's VLAN 20 client. Why?
Inter-VLAN routing requires L3. SW2 is a pure L2 switch — it only forwards within the same VLAN. To reach VLAN 20, the frame must go to SW1, be routed via SW1's SVI (10.20.0.1), and returned. KK identified Karl Rule: SW1 must have ipforwarding enabled on the Default VLAN (uplink) or the return packet drops.
Q2
What is LLDP and what does it tell you?
Link Layer Discovery Protocol — L2 neighbour discovery. Each device advertises its hostname, port, capabilities, and management IP. Run show lldp neighbours on EXOS to see what's connected on each port. Critical for verifying trunk neighbours and AP uplinks.
Q3–4
Why must VLAN 1 be removed from Port 10 on both switches? What's the risk?
If VLAN 1 (Default) is present untagged or tagged on both ends of the trunk, and also used as the management/uplink VLAN on both switches, you create a bridging loop. Broadcasts circulate endlessly. EXOS: configure vlan Default delete ports 10 on BOTH switches.
Q5–8
MAC address aging, FDB miss after aging, native VLAN on trunk, port 10 tagging verification
FDB entries age out (default 300s on EXOS). After aging, the next frame is flooded again. Native/default VLAN on trunk determines which VLAN gets untagged frames — must match on both ends. Verify with show vlan and show ports 10 information detail.
Session 3

L3 Routing — IPv4, SVIs, Static vs Dynamic, Campus Design · Apr 22 · 8 Questions

Q1
What is a /24 subnet and how many hosts does it support?
/24 = 255.255.255.0. 256 addresses, 254 usable hosts (subtract network + broadcast). In the lab: 10.10.0.0/24 → hosts 10.10.0.1 to 10.10.0.254. KK correctly derived host range from the mask.
Q2
What is an SVI and how does it act as a default gateway?
SVI (Switched Virtual Interface) = a virtual L3 interface bound to a VLAN on a switch. It has an IP address that becomes the default gateway for that VLAN's hosts. In the lab: SW1 has SVI 10.10.0.1 (VLAN 10), 10.20.0.1 (VLAN 20), 10.30.0.1 (VLAN 30). Clients point to these as their gateway.
Q3
Why does SW2 NOT have any SVIs?
SW2 is a pure L2 switch in this architecture. All routing happens on SW1. Giving SW2 SVIs would create routing conflicts and complexity. KK identified this from the lab architecture memory.
Q4
Static route vs OSPF — when do you use each?
Static route: manually configured, deterministic, no overhead. Use for small/stable networks or specific paths. OSPF: dynamic routing protocol, discovers neighbours, exchanges topology, finds best path automatically. Use for larger networks or redundant paths where static routes become unmanageable.
Q5
The Quantum Fiber modem needs to know about your VLANs. How?
Add static routes on the modem pointing to SW1's uplink IP. In the lab: three routes added — 10.10.0.0/24 → 192.168.0.28, 10.20.0.0/24 → 192.168.0.28, 10.30.0.0/24 → 192.168.0.28. Without this, the modem drops return traffic (it doesn't know those subnets exist).
Q6–8
ARP process, campus network hierarchy, /25 subnetting exercise
ARP: client broadcasts "who has 10.10.0.1?" → gateway responds with its MAC → client caches the ARP entry and sends frames to that MAC. Campus hierarchy: Access layer (client ports) → Distribution (VLAN aggregation + routing) → Core (high-speed backbone). /25 splits a /24 in half: 10.10.0.0/25 (hosts 1-126) + 10.10.0.128/25 (hosts 129-254).
Session 4

Wireless — RF, Channels, Auth, WPA Simulator · Apr 22 · 8 Questions

Q1
A client shows RSSI of -72 dBm. Is that good or bad?
-72 dBm is moderate — adequate for basic connectivity but not ideal. Scale: -50 dBm = excellent, -70 dBm = acceptable, -80 dBm = marginal, -90 dBm = unusable. Negative scale: closer to zero is stronger.
Q2
2.4 GHz vs 5 GHz — when would you use each?
2.4 GHz: longer range, better penetration through walls. 3 non-overlapping channels (1, 6, 11). Crowded (IoT, neighbours). Use for range-sensitive devices or legacy clients. 5 GHz: shorter range, much higher throughput. Many channels (including DFS). Use for high-bandwidth clients close to AP.
Q3
What is a DFS channel and why does it matter?
Dynamic Frequency Selection. Channels on UNII-2A (100-140 MHz) and UNII-2C that may be used by radar (weather, military). If an AP detects radar, it must vacate the channel within 10 seconds. Causes brief disconnection. XIQ RRM can handle DFS avoidance automatically.
Q4
Walk through the WPA Simulator — what happens between client and AP from power-on to browsing?
Full lifecycle (used WPA Simulator in Session 4):
1. Discovery: AP broadcasts Beacon with SSID/capabilities. Client sends Probe Request → AP responds Probe Response.
2. Authentication: WPA3 SAE = Dragonfly handshake. No password transmitted — mutual derivation of PMK. Forward secrecy: even if password later compromised, past sessions unreadable.
3. Association: Client and AP exchange capabilities (rates, HT/VHT/HE mode, QoS support).
4. 4-Way Handshake: M1: AP sends ANonce. M2: Client sends SNonce + MIC (proves key knowledge). M3: AP sends GTK (group key) encrypted + MIC. M4: Client ACKs. Both sides derive PTK (Pairwise Transient Key) from ANonce+SNonce+PMK.
5. Data Plane: DHCP (client → broadcast → server → offer → request → ack) → DNS → TCP 3-way → Zoom stream.
Encryption boundary: WiFi L2 frame payload encrypted with PTK from client NIC to AP antenna. Wired side = unencrypted.
Q5
WPA2 PSK weakness vs WPA3 SAE?
WPA2 PSK: the 4-way handshake can be captured by a passive listener. The PSK can be recovered by offline dictionary attack on the captured handshake. WPA3 SAE (Dragonfly): the password is never transmitted. Each session derives a unique PMK — even if you capture the handshake and later learn the password, you cannot decrypt past sessions (forward secrecy).
Q6
What is the difference between authentication and association in 802.11?
Authentication: client proves it knows the credentials / passes 802.1X. Association: client negotiates parameters (rates, MIMO streams, QoS) and gets an Association ID (AID) — the logical slot at the AP. Both required before data flows.
Q7–8
WiFi 6 vs WiFi 7 MLO, 802.1X on wired port 5
WiFi 6 (802.11ax): OFDMA (divide channel into subcarriers for multiple clients), MU-MIMO (spatial streams), TWT (Target Wake Time for IoT battery). WiFi 7 (802.11be): MLO — Multi-Link Operation. Client simultaneously uses 2.4+5+6 GHz. Load balances and failovers between bands transparently. 802.1X on wired Port 5 (VLAN 10): currently not configured in lab = open access. Should add port-based 802.1X for full enterprise hardening.

XIQ Platform — AP Onboarding, Navigation, Troubleshooting · Apr 22 · 7 Questions

Q1
Walk me through how an AP comes online in XIQ from factory state.
Pre-register
Enter AP serial number in XIQ portal. Assign to device group / location.
Boot on Default VLAN
AP boots on untagged port. Gets DHCP from Default VLAN (SW1). Karl Rule + AP Lifeline must be in place.
DNS Redirector
AP resolves Aerohive/Extreme redirector DNS. Cloud redirects to XIQ tenant.
XIQ Onboard
AP appears in XIQ as pending. Matches pre-registered serial. Assigned Network Policy.
Policy Push
XIQ pushes SSIDs, VLANs, QoS, firewall rules, channel/power settings to AP. AP broadcasts SSIDs.
Q2
Where in XIQ do you see all currently connected clients?
Monitor → Clients. Shows: client hostname/MAC, signal (RSSI), AP it's connected to, VLAN, IP address, data rates, roam history. Filter by SSID, VLAN, or AP to narrow down.
Q3
A client shows "Connected" in XIQ but has no internet. Walk me through your first 5 checks.
1. XIQ Monitor: confirm client has an IP (DHCP success) and which VLAN. 2. Check AP port on SW1/SW2 — correct VLANs tagged? 3. SSH to SW1: show iparp — is client in ARP table? 4. Check SW1 SVI: show vlan VLAN_30 — is ipforwarding enabled? 5. Check Karl Rule: show ipforwarding — Default VLAN must show "enabled". That's the most common miss.
Q4–7
Network Policy push workflow, User Profile / firewall rule navigation, QoS weights location, AP radio power adjustment
Network Policy: XIQ → Configure → Network Policies → select policy → Deploy. QoS weights: XIQ → User Profiles → select profile → Scheduling tab → Weight value. Radio power: XIQ → Devices → AP → Radio tab → Tx Power override. Firewall rules: XIQ → User Profiles → select profile → Firewall tab → Add Rule → select IP objects (must exist in Common Objects first).

Troubleshooting + Packet Capture — L1→L2→L3 · Apr 22 · 8 Questions

Scenario
VLAN 30 Guest client associates to AP, gets IP 10.30.0.150/24, shows "Connected" — but cannot ping 8.8.8.8 or browse. Walk through the full diagnosis.
L1
What are your Layer 1 checks?
Port LED status on switch. show ports 3 information — link state UP/DOWN. Physical cable check on wired segments. AP power (PoE): show inline-power on port 3. Confirm AP is actually broadcasting the SSID (not just associated in XIQ).
L2
Layer 2 checks — what are you verifying?
Port 3 VLAN tagging: VLAN 30 must be tagged on Port 3. show vlan VLAN_30 — is port 3 listed as tagged? Trunk port 10: VLAN 30 must pass through to SW1. show fdb vlan VLAN_30 — does the client MAC appear? If MAC not in FDB, traffic not reaching SW1.
L3
Layer 3 checks — you've confirmed L1 and L2 are fine. What next?
DHCP lease: show dhcp-server lease vlan VLAN_30 — client 10.30.0.150 should appear. SVI: ping 10.30.0.1 from SW1 — gateway should respond. ipforwarding: show ipforwarding — ALL VLANs including Default must show enabled. If Default shows disabled → Karl Rule violation → internet drops for all clients.
Q4
Root cause identified: Karl Rule. What was the actual packet path that failed?
Client sends packet to 8.8.8.8. SW1 routes it out Port 1 to Quantum Fiber modem (192.168.0.1). Modem forwards to internet. Response arrives back at Port 1 (Default VLAN, untagged). SW1 receives the return packet on Default VLAN. If ipforwarding is disabled on Default VLAN → SW1 checks routing table, sees routing disabled on that interface, drops the packet. Client never gets the reply. Symptom: one-way traffic — ICMP request leaves but no echo reply returns.
Q5
Traceroute vs packet capture — when do you use each?
Traceroute: shows L3 path (hop-by-hop routers to destination). Tells you WHERE the path breaks (which hop stops responding). Packet capture: shows actual frame content (L2 headers, payload, flags). Tells you WHAT the packet contains. Use traceroute first to identify the failure hop. Use packet capture to confirm packet content or detect malformed/dropped frames.
Q6–8
Wireshark capture points, ICMP filter, interpreting capture output
Capture points in the lab:
Port 1 (uplink): Capture here to see if the ICMP Echo Reply comes back from modem. If you see the Echo Request leave but no Reply returns → modem/ISP issue.
Port 3 (AP port): Capture here to see client traffic. If Echo Request doesn't appear on Port 3 at all → client is not transmitting (radio or association issue).

Filter: ip.dst == 10.30.0.150 — shows all traffic addressed TO the guest client. Look for ICMP Echo Reply (type 0). If absent → packet dropped before reaching client.

Karl Rule indicator in capture: Echo Request arrives at Port 1 heading out. Echo Reply arrives BACK at Port 1. Reply is for 10.30.0.150 (Default VLAN). No forwarding on Default → Reply disappears. You see request leave, reply arrive, reply vanish. That's the Karl Rule signature.
