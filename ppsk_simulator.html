Protocol & RFC Reference

IEEE 802.11-2020 ↗
Wi-Fi MAC + PHY Standard
Defines Association, Authentication, Beacon frames, RSN IE, 4-Way Handshake (§12.7.6), CCMP/AES encryption (§12.5.3). All PPSK Wi-Fi framing.
IEEE 802.11i (2004) ↗
RSN — WPA2 Security Amendment
Defines the Robust Security Network (RSN) framework: PMK derivation, 4-Way Handshake key hierarchy (PTK = KCK+KEK+TK), GTK distribution. Now folded into 802.11-2020.
RFC 2898 / PKCS#5 ↗
PBKDF2 — Password-Based Key Derivation
PPSK PMK derivation: PMK = PBKDF2(HMAC-SHA1, passphrase, SSID, 4096, 256-bit). Salt = SSID name per 802.11i §8.5.1.3. Identical derivation at XIQ (provisioning) and UE (runtime).
RFC 5415 ↗
CAPWAP — Control and Provisioning of Wireless APs
AP ↔ XIQ management tunnel. UDP 5246 (control), 5247 (data). Carries SSID config, User Group definitions, RadSec certificate push. Always running — not per-auth.
RFC 6614 ↗
RadSec — RADIUS over TLS
AP1 calls XIQ per-auth on TCP 2083. RADIUS framing inside a TLS tunnel. Returns PMK + VLAN + firewall profile. This is the PPSK runtime validation channel — replaces UDP RADIUS with encrypted TCP.
RFC 2865 ↗
RADIUS — Remote Auth Dial-In User Service
Access-Request / Access-Accept payload structure. Attr 1 (User-Name), Attr 79 (EAP-Message), Attr 80 (Message-Authenticator). Same framing inside RadSec TLS tunnel.
RFC 2548 ↗
Microsoft Vendor-Specific RADIUS Attributes
MS-MPPE-Recv-Key (Attr 26, Vendor 311, Type 17) — carries PMK from XIQ to AP in Access-Accept. Same attribute format used in 802.1X/NPS to deliver MSK. PPSK reuses this payload.
RFC 2868 ↗
RADIUS Tunnel Attributes
Tunnel-Type (Attr 64) = VLAN(13), Tunnel-Medium-Type (Attr 65) = IEEE 802(6), Tunnel-Private-Group-ID (Attr 81) = "80". Same as 802.1X VLAN assignment. AP1 reads these from Access-Accept.
RFC 2131 ↗
DHCP — Dynamic Host Configuration Protocol
4-message exchange: Discover → Offer → Request → ACK. Client gets 10.80.0.x from SW1 VLAN pool. Lease = 86400s (24h — matches PPSK key validity). Smoking-gun confirmation of E2E success.
IEEE 802.1Q ↗
VLAN Tagging Standard
4-byte VLAN tag in Ethernet frame header. AP1 adds VLAN tag to all guest frames after M4 ACK. SW1 EXOS forwards on tagged trunk port. Firewall applied before tag is added.
IEEE 802.1X ↗
Port-Based Network Access Control
Controlled port model: port locked until auth complete. PPSK uses the same model — EAPOL 4-Way Handshake, port open after M4 ACK, VLAN from Access-Accept Attr 81. No EAP exchange needed.
