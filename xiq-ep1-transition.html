From XIQ to Extreme Platform One: A Complete Practitioner’s Transition Guide

ExtremeCloud IQ (XIQ) GUI is being retired on July 1, 2026. Extreme Platform One (EP1) is its replacement — a next-generation portal with a redesigned navigation model, new terminology, and reorganised workflows. This guide documents every feature area exhaustively: where it lived in XIQ, where it lives in EP1, what changed, and what gaps exist. Built as a practitioner’s reference for guiding clients through the transition.

Hard Deadline

July 1, 2026 — XIQ GUI removed. All clients must be operating in EP1 by this date. This guide must be client-ready before June 26 to allow one week of review buffer. 47 days from sprint start.

What Is Extreme Platform One (EP1)?

EP1 is the next-generation Extreme Networks cloud management portal, accessed via the same login as XIQ but via the 9-dot launcher in the top navigation. It is not a new product — it is a new UI framework replacing the current XIQ web app with a unified platform that also integrates SD-WAN (IPE), fabric management, and analytics under one roof.

DimensionXIQ (current)EP1 (replacement)
Accessextremecloudiq.comSame login → 9-dot launcher → EP1 app
Navigation modelTop tabs: Monitor / Configure / DevicesLeft sidebar + module-based launcher
TerminologyNetwork Policies, Common ObjectsRenamed — to be documented per week
Deploy impactXIQ GUI removed July 1EP1 is the only portal after July 1
APIXIQ API (unchanged)Same underlying API
When EP1 is downN/ADeploys blocked, PPSK RadSec affected — check Extreme cloud status page

Known from Lab Experience

EP1 downtime blocks ALL policy deploys and PPSK RadSec authentication. If clients report devices not updating or PPSK failing after July 1, check the Extreme cloud status page before diving into switch/AP config. This was the May 14 lesson — what looks like a local CAPWAP issue can be an EP1 cloud outage.
