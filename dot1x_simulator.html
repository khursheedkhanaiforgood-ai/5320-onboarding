Acronym Decoder

EAPOL
EAP Over LAN
Thin wrapper that carries EAP messages over Ethernet/Wi-Fi. The envelope.
EAP
Extensible Authentication Protocol
The conversation format for auth. Not a method itself — PEAP/TLS/MSCHAPv2 plug into it.
PEAP
Protected EAP
Wraps EAP inside a TLS tunnel. Like HTTPS for auth. Hides the password exchange from OTA capture.
MSCHAPv2
Microsoft CHAPv2
Password verification inside the PEAP tunnel. Challenge-response using NT-Hash — password never in plaintext.
RADIUS
Remote Auth Dial-In User Service
Protocol AP uses to talk to NPS. UDP port 1812. AP is the proxy — client has no direct NPS connection.
NPS
Network Policy Server
Microsoft's RADIUS server. Validates users against AD. Assigns VLAN via Network Policy rules.
MSK
Master Session Key
512-bit key generated by NPS from TLS + MSCHAPv2 material. Sent to AP via RADIUS. Never travels OTA.
PMK
Pairwise Master Key
Root key for 4-way handshake. = first 256 bits of MSK (Enterprise) or PBKDF2(passphrase) (Personal).
PTK
Pairwise Transient Key
Session key derived from PMK + nonces + MACs. Split into KCK + KEK + TK.
TK
Temporal Key
The actual encryption key for unicast data frames (CCMP/AES). Unique per client per session.
GTK
Group Temporal Key
Encrypts broadcast/multicast frames. Shared by all clients. Rotation = May 8 incident cause.
ANonce / SNonce
AP Nonce / Station Nonce
Fresh random values generated each session. Combined with PMK + MACs to derive PTK. Nonce = "number used once".
MIC
Message Integrity Code
Cryptographic signature on handshake messages. Computed with KCK. Proves you hold the correct PMK.
KCK / KEK
Key Confirmation Key / Key Encryption Key
KCK signs the MIC in M2/M3/M4. KEK encrypts the GTK inside M3.

PMK Source — Personal vs Enterprise

WPA-Personal (PSK)

PMK = PBKDF2(passphrase + SSID, 4096, 256)

Same PMK for every client, every session. Capture M1+M2 OTA → run dictionary attack → derive PTK → decrypt all traffic. May 8 GTK rotation affected all clients because PMK is shared.

WPA-Enterprise (802.1X)

PMK = MSK[:256] (unique per session)

MSK generated from TLS + MSCHAPv2 material. Never travels OTA. Even with a captured handshake, no offline attack is possible. Each client gets its own PMK — GTK rotation still applies but PTK is per-client.
