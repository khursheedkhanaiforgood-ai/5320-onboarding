E2E Key Learnings

Each lesson from this lab mapped to the underlying networking concept. Click any entry to jump to the section where it was applied, or follow the standard citation for deeper study.

L3 Routing
ipforwarding must be enabled on every routed VLAN — including Default
Applied in: Lab Phase 1 (SW1) · Lab Phase 2 (SW2) · Session Log §VLAN config · Onboarding §EXOS setup  —  RFC 1812 ↗
NAT
Quantum Fiber needs static routes back to your private subnets
Applied in: Lab Phase 1 (SW1 routes) · Lab Phase 2 (SW2 routes) · Session Log §Internet verify · Session Log §Phase 2 troubleshoot  —  RFC 3022 ↗
802.1Q
A port is untagged on one VLAN but tagged on many
Applied in: Lab LLD (VLAN/subnet plan) · Session Log §VLAN build · Lab Phase 1 exec · Onboarding §EXOS setup  —  IEEE 802.1Q ↗
DHCP
The switch is the DHCP server — gateway, DNS, and range configured on the switch itself
Applied in: Lab Phase 1 (SW1 DHCP) · Lab Phase 2 (SW2 DHCP) · Session Log §VLAN config · Onboarding §Command ref  —  RFC 2131 ↗
ARP
show iparp shows neighbors — show iproute shows the switch’s own addresses
Applied in: Session Log §Phase 2 (“two Default”) · Session Log §Phase 2 troubleshoot · Lab Phase 2 verify steps  —  RFC 826 ↗
PoE
Power and data on the same cable — switch negotiates class with the AP
Applied in: Lab Phase 1 (enable inline-power) · Lab Phase 2 · Session Log §AP onboarding  —  IEEE 802.3af ↗
CAPWAP
AP3000 SSID config is cloud-only — no AP CLI for SSIDs
Applied in: Session Log §AP3000-A onboarding · Session Log §AP3000-B (Phase 2) · Lab Phase 1 AP step · Onboarding §XIQ confirmation  —  RFC 5415 ↗
