VOSS Concepts Learned Today

1. I-SID — The "Service Suitcase"

In EXOS, a VLAN is just a VLAN. In VOSS, every VLAN that crosses the fabric gets a 24-bit I-SID (Service Instance Identifier) — like a VNI in VXLAN. The I-SID is the global identity of the service across all fabric nodes.

Convention in this lab: I-SID = VLAN number + 100,000. So VLAN 50 → I-SID 100050.

Command: vlan i-sid 50 100050 — simpler than the manual i-sid elan c-vid approach.

2. Fabric Attach (FA) — Auto-Provisioning

AP2 already speaks FA (confirmed at boot: clientWapType1 on port 1/3). When VIQ pushes Corp/Guest SSIDs to AP2, the AP sends IEEE 802.1Qcj LLDP TLVs to SW2:

"I need VLAN 50 (I-SID 100050) on this port"
"I need VLAN 60 (I-SID 100060) on this port"

SW2 auto-approves and dynamically tags port 1/3 with those VLANs. You never manually tag the AP port.

3. Greenfield / ZTD State

The "A" (Auto-sense) origin on I-SID 15999999 means the switch has auto-discovered ports but no production services are defined. This is the clean starting point before Corp/Guest services are added.

4. IP Shortcuts (for next session)

Enabling ip-shortcut under router isis turns SW2 into a "Service Gateway" — it advertises its IP routes (including the default route via IPE) through the SPB fabric. Other fabric nodes automatically learn the internet exit without needing static routes.
