⎯⎯ E2E Key Learnings

Each lesson from this lab mapped to the underlying networking concept. Click to jump to the relevant guide section or standard.

  1. L3 Routing
    ipforwarding must be enabled on every routed VLAN — including Default — missing on Default breaks internet for all clients.
    Applied in: Lab Phase 1 (SW1) · Lab Phase 2 (SW2) · Session Log §VLAN config · Onboarding Guide §EXOS setup  |  RFC 1812 deep-dive ↗
  2. NAT
    Quantum Fiber needs static routes back to your private subnets — without them, internet replies are silently dropped.
    Applied in: Lab Phase 1 (SW1 routes) · Lab Phase 2 (SW2 routes) · Session Log §Internet verify · Session Log §Phase 2 troubleshoot  |  RFC 3022 deep-dive ↗
  3. 802.1Q
    A port is untagged on one VLAN but tagged on many — AP trunk carries Corp + Guest SSIDs on one cable.
    Applied in: Lab LLD (VLAN/subnet plan) · Session Log §VLAN build · Lab Phase 1 exec · Onboarding Guide §EXOS setup  |  IEEE 802.1Q deep-dive ↗
  4. DHCP
    The switch is the DHCP server — gateway, DNS, and range configured on the switch itself.
    Applied in: Lab Phase 1 (SW1 DHCP) · Lab Phase 2 (SW2 DHCP) · Session Log §VLAN config · Onboarding §Command ref  |  RFC 2131 deep-dive ↗
  5. ARP
    show iparp shows neighbors — show iproute shows the switch's own addresses. A device never ARPs for itself.
    Applied in: Session Log §Phase 2 ("two Default") · Session Log §Phase 2 troubleshoot · Lab Phase 2 verify steps  |  RFC 826 deep-dive ↗
  6. PoE
    Power and data on the same cable — switch negotiates class with the AP. AP3000 = Class 4, 5.2W at idle.
    Applied in: Lab Phase 1 (enable inline-power) · Lab Phase 2 · Session Log §AP onboarding  |  IEEE 802.3af deep-dive ↗
  7. CAPWAP
    AP3000 SSID config is cloud-only — no AP CLI for SSIDs. XIQ "Reset & Delete" clears org binding; plain "Delete" does not.
    Applied in: Session Log §AP3000-A onboarding · Session Log §AP3000-B (Phase 2) · Lab Phase 1 AP step · Onboarding §XIQ confirmation  |  RFC 5415 deep-dive ↗
  8. AI Agent
    State machine + LLM = domain-specific autonomous agent. Regex detects state cheaply; Claude API called only on transitions. Read-only on the wire.
    Applied in: Agent Tutorial §Module map · Agent Tutorial §20-state machine · Deploy Guide (Local + Railway) · Railway Tutorial  |  AI Agents deep-dive ↗
