E2E Key Learnings — Concepts Every Networking Student Should Know

Each lesson from this lab mapped to the underlying networking concept and where to go deeper. These are the moments where something broke, was debugged, and truly understood.

L3 Routing
ipforwarding must be enabled on every routed VLAN — including Default
Omitting it on Default VLAN broke internet for all WiFi clients even though Corp/Guest VLANs were enabled. The switch is an L3 device — each VLAN interface must explicitly opt in to forwarding. Concept: directly connected routes and per-interface forwarding decisions. See: RFC 1812 §5.2, Bertsekas & Gallager Ch. 5.
NAT / Return Path
Quantum Fiber needs static routes to reach your private subnets
Quantum Fiber NATs outbound traffic — your 10.20.x.x client reaches 8.8.8.8, but the reply comes back to Quantum Fiber which has no route for 10.20.0.0/24. Without static routes pointing to the switch, replies are silently dropped. Concept: NAT, return-path routing, RFC 1918 private addressing. See: RFC 3022 (NAT), RFC 1918.
802.1Q VLANs
A port can be untagged on only one VLAN — but tagged on many
The AP3000 carries Corp1 (VLAN 20) and Guest1 (VLAN 30) over a single cable using 802.1Q tags. The switch port is untagged on Default (for AP→XIQ provisioning) and tagged on Corp1+Guest1 (for SSID traffic). Concept: trunk ports, access ports, VLAN tagging, 4-byte 802.1Q header. See: IEEE 802.1Q-2018.
DHCP
The switch itself is the DHCP server — not the AP, not the cloud
On EXOS, enable dhcp ports 3 vlan Corp1 starts the switch's built-in DHCP server on that VLAN. The gateway, DNS, and address range are configured directly on the switch. Concept: DORA sequence, DHCP relay, gateway and DNS options. See: RFC 2131.
ARP vs Routing
show iparp shows neighbors — show iproute shows the switch's own addresses
A device never ARPs for itself. The ARP table contains only remote devices the switch has discovered. The route table contains the switch's own directly-connected networks and learned routes. Confusing these two is a classic beginner error. Concept: ARP, MAC-to-IP resolution, routing table vs ARP cache. See: RFC 826 (ARP), RFC 1812.
PoE / 802.3af
Power and data on the same cable — the switch negotiates class with the AP
The EN-5320 detects the AP3000 as a Class 4 device (up to 30W) and delivers 5.2W at idle. No separate power supply needed. show inline-power info ports 3 shows voltage, current, and watts in real time. Concept: IEEE 802.3af/at/bt power classes, detection handshake.
CAPWAP / XIQ
AP3000 SSID config is cloud-only — there is no AP CLI for SSID configuration
The AP3000 uses CAPWAP to tunnel management traffic to XIQ. All SSID, radio, and VLAN mapping config is pushed from the cloud. The AP CLI is limited to diagnostics. "Reset and Delete" in XIQ clears the org binding on the AP — "Delete" alone does not. Concept: CAPWAP split-MAC architecture, cloud controller model. See: RFC 5415, RFC 5416.
AI Agents
A state machine + LLM = a domain-specific autonomous agent
The 5320 onboarding agent combines a 20-state regex machine (cheap, local, fast) with Claude API calls only on state transitions (expensive, cloud, powerful). The serial port is read-only — the human always types the commands. This separation of concerns (observe vs act) is a core agentic design principle. See: Agent Build Tutorial, Anthropic Agents docs.
