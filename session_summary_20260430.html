XIQ DNS Issue — Root Cause Analysis

IQAgent could not resolve hac.extremecloudiq.com despite ip name-server primary 8.8.8.8 being configured. 0 DNS requests ever sent.

What Happened

  • VOSS booted → cloud agent tried ZTP+ → failed (no DNS at that point — correct)
  • We configured static IP + DNS → restarted IQAgent
  • IQAgent still showed "DNS Unable to resolve [hac.extremecloudiq.com]"
  • show ip dns → Status: Inactive, Total requests: 0
  • Tried direct IP (34.253.190.222) → HTTPS connection failed: "Unable to get management IP"
  • Root cause: TLS cert is issued for hostname, not IP. Connection fails at handshake.

Root Cause

The VOSS ip name-server primary 8.8.8.8 CLI command configures DNS for the VOSS management plane, but the IQAgent (HiveAgent) runs as a Linux-level process and likely reads /etc/resolv.conf — which is only populated by DHCP at boot, not by the CLI command.


When ZTP+ gets DHCP at boot, the DHCP lease includes DNS servers which go into /etc/resolv.conf. With a manual static IP, no DHCP lease = no /etc/resolv.conf = IQAgent can't resolve hostnames.

Things Tried (all failed)

iqagent server hac.extremecloudiq.com   ← DNS still fails
iqagent server 34.253.190.222           ← TLS cert mismatch
no ip address + ip address dhcp         ← VOSS VLAN interface doesn't support DHCP client
bash / debug bash enable                ← not accessible from standard CLI

Tomorrow — Fresh Start Plan

User decision: wipe SW2 completely and start clean. Correct call — the manual bootstrap created a messy state.

Step 1 — Wipe SW2 (VOSS factory reset)

-- From VOSS CLI --
enable
delete /intflash/config.cfg
reload
VOSS reset = delete /intflash/config.cfg — NOT unconfigure switch all (that's EXOS). Don't confuse them.

After reload: SW2 boots clean. rwa/rwa default restored.

Step 2 — Pre-Wire BEFORE Boot

Before powering on SW2, connect Port 1 to HomeModem LAN port. This ensures DHCP runs at boot and ZTP+ gets DNS automatically from HomeModem's DHCP pool.

HomeModem DHCP gives: IP + subnet + gateway + DNS. ZTP+ uses that DNS to resolve hac.extremecloudiq.com. No manual ip name-server needed.

Step 3 — Let ZTP+ Do the Work

SW2 boots → ZTP+ starts → DHCP on Port 1 → gets IP from HomeModem → DNS resolves XIQ → cloud agent connects → SW2 appears in XIQ Inbox as Fabric Engine device.


In XIQ: Accept device from Inbox → assign a VOSS/Fabric Engine network policy → push IS-IS + SPB + Nickname 0.00.02 + VLANs from XIQ.

Delete the old EXOS SW2 entry from XIQ inventory BEFORE boot so it comes in clean.

Step 4 — After XIQ Connects, Set Static IP

Once XIQ is managing SW2, configure a static management IP (192.168.0.31) via XIQ device template. Then Port 1 can be reconfigured as the Transit Port to IPE (10.0.1.1/30).
