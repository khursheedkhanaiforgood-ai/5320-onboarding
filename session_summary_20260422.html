Full Client Lifecycle: Discovery (Beacon/Probe) → Auth (SAE Dragonfly handshake: no password transmitted, forward secrecy, mutual auth) → Association (capabilities exchange) → 4-Way Handshake (M1: ANonce from AP / M2: SNonce from client + MIC / M3: GTK encrypted + MIC / M4: ACK) → Data Plane (DHCP → DNS → TCP → Zoom).

WiFi 6 vs 7: WiFi 6 = OFDMA + MU-MIMO + TWT. WiFi 7 = MLO (Multi-Link Operation) — client can aggregate 2.4+5+6 GHz simultaneously.

Apr 22

Karl's Lab Debrief — 15 Slides (NEW)

Karl's Lab_Apr 21_Horizon_Khursheed's Debrief.pptx — first ingested Apr 22. Visual synthesis of the full lab build with field notes, CLI, and the Alex closing quote.
Slide 1
Goal: Primary/Secondary 5320 Architecture
Enterprise branch in a single afternoon. SW1 = L3 Core, SW2 = L2 Access. Prove the architecture before rolling out to customer sites.
Slides 2–3
Anatomy — Physical + Logical Topology
SW1 port map: Port 1 = uplink, Port 3 = AP (Default untagged, VLAN 20+30 tagged), Port 5 = wired client (VLAN 10 untagged), Port 10 = trunk. SW2 = mirror access ports only. VLAN table: 10/20/30 → 10.x.0.0/24 subnets.
Slide 4
Port 10 Trunk — VLAN 1 Removal
VLAN 1 must be explicitly removed from trunk port 10 on BOTH switches to prevent bridging loop. configure vlan Default delete ports 10 — run on both.
Slide 5
DHCP Reach-Back — Port 10 Must Be Included
SW2 clients need DHCP from SW1. DHCP helper must include port 10 in the DHCP scope so reach-back frames traverse the trunk.
enable dhcp ports 3,10 vlan VLAN_10
enable dhcp ports 3,10 vlan VLAN_20
enable dhcp ports 3,10 vlan VLAN_30
Slides 6–7
Internet Ghosting → Karl Rule Explained
Without ipforwarding on Default VLAN: "Return traffic hits the Default VLAN (modem uplink). The switch checks the Default VLAN state, sees routing is disabled, and explicitly drops the packet." Client gets DHCP but zero internet — the ghost symptom.
enable ipforwarding vlan Default   ← KARL RULE — run this first
Slides 8–9
QoS Strategy + E2E Execution
CLI: VLAN 10/20 → qp6 (high priority), VLAN 30 → qp1 (low). XIQ: Corp_20 User Profile → Scheduling Weight 100, Guest_30 → Weight 10. Applied at both SW1 edge and AP radio level for true end-to-end QoS.
Slides 10–11
Threat Model + Defense Matrix
North-South: XIQ L3 Firewall Rules in Guest_30 User Profile — DENY 10.30.0.0/24 → 10.10.0.0/24, DENY 10.30.0.0/24 → 10.20.0.0/24.
East-West (intra-VLAN): XIQ Station Isolation — "Drop traffic between stations" action in Guest_30 firewall profile. Cannot be done with EXOS ACLs alone (L2 intra-VLAN traffic bypasses ACL engine).
Slide 12
EXOS ACL Syntax Field Notes
Cisco-style one-liners FAIL on EXOS. Must use the policy editor with entry containers.
edit policy GUEST_ISOLATION          # opens vi editor
entry deny-to-corp {
  if {
    source-address 10.30.0.0/24;
    destination-address 10.10.0.0/24;
  } then {
    deny;
  }
}
entry permit-all { if { } then { permit; } }
# Esc + :wq to save. For this lab: bypassed via XIQ.
Slide 13
OWE Transition Mode
AP broadcasts TWO SSIDs simultaneously: (1) visible open transition SSID for legacy devices, (2) hidden secure OWE SSID. OWE-capable devices auto-migrate to the secure SSID. XIQ toggle: Guest_Wireless SSID → "Transition Mode for 2.4GHz and 5GHz".
Slide 14–15
Blueprint + Retrospective
The lab proved the Primary/Secondary architecture end-to-end. All security layers operational. The final Alex note:
"By combining the raw routing power of the 5320s with the edge intelligence of the AP3000s, we built an enterprise-grade branch office in a single afternoon."
— Alex, slide 15 retrospective

8 Key Lessons Locked

1
Karl Rule
enable ipforwarding vlan Default on SW1. Without it: clients get IPs, no internet. Return traffic dropped at unrouted Default VLAN.
2
DHCP Reach-Back
enable dhcp ports 5,10 vlan VLAN_10 — port 10 MUST be included for SW2 clients
3
VLAN 1 Triangle Loop
configure vlan Default delete ports 10 on BOTH switches. Loop prevention.
4
AP Lifeline
configure vlan Default add ports 3 untagged — AP boots dumb, needs untagged VLAN 1 to reach modem and call home to XIQ
5
EXOS ACL Syntax
edit policy NAME (vi editor). Use source-address keyword, entry containers with semicolons, explicit permit all at end. Cisco one-liners FAIL.
6
Guest-to-Guest Isolation
ACLs cannot stop L2 intra-VLAN traffic. Must use XIQ "Drop traffic between stations" firewall action on AP.
7
OWE Transition Mode
Enable toggle in XIQ Guest SSID — runs hidden open SSID alongside OWE SSID for legacy device backward compat.
8
XIQ IP Objects
Must create in Common Objects BEFORE adding firewall rules, or "IP object must be saved first" error blocks you.
