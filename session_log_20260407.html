10-Step XIQ Deploy — Linked List

Step 3
VLAN Attributes. Added VLANs 20/30/50/60. Hiccup: table appeared empty — root cause: switches not yet assigned to Test_SW1 policy.
Step 4
Network Allocation. Four subnets defined. SVIs auto-set (.1 of each). Hiccup: adding VLAN 1 to Routing section broke XIQ↔switch comms. Rule: never add VLAN 1 to Routing section.
Step 5
Routing — SVIs pushed by XIQ. IPv4 Forwarding enabled. Confirmed via show vlan — f flag + 10.20.0.1/10.30.0.1 present. DHCP + default route NOT pushed.
Step 6
Static Routes — BLOCKED. XIQ error: "next hop IP doesn't match any subnetworks." Root cause confirmed by Extreme expert: XIQ cannot push L3 static routes via L2 interface. VLAN 1 not visible as managed subnet. Deferred to CLI.
Step 7
Switch access failures. SSH failed (XIQ ZTP+ set unknown password). Console login failed (same). XIQ CLI script push — no output (SSH issue XIQ side). Eventually got in via console.
Step 8
Port Types — discovered missing config. VLANs 20/30 showed 0/0 ports. Created AP_Port: Trunk, Native VLAN 1, Tagged 20/30/50/60, Edge Port ON, PoE ON. Assigned port 3 both switches. Pushed — 1/1 ports confirmed.
Step 9
DHCP via CLI. Correct syntax required (see lessons below). iPhone got 10.20.0.x but no internet — missing enable ipforwarding vlan Default.

⚠ Critical

Lesson 1 — enable ipforwarding vlan Default

⚠ Mandatory after every factory reset — not set by XIQ

Return traffic from the internet arrives on VLAN 1 (Default) — the uplink to the Quantum Fiber router. Without IP forwarding enabled on Default VLAN, clients get DHCP and a gateway but Safari times out. Must be done via CLI after every factory reset.

enable ipforwarding vlan Default
save configuration

Lesson 3 — Default Route is Automatic via DHCP

⚠ Do NOT manually add the default route

EXOS installs a BOOTP default route automatically when the switch gets its management IP via DHCP. Verify with show iproute — look for origin bo (BOOTP). Only enable ipforwarding vlan Default is manual.

VLAN 10 Execution

VLAN 10 Management — What Happened April 7

XIQ
VLAN 10 created in XIQ. VLAN Attribute: 10 / SW_VLAN_10_Mgmt. Network Allocation: SW_Mgmt → 10.10.0.0/24.
XIQ
Routing per device configured. SW1 (FJ012544G-00483): 10.10.0.1/24. SW2 (FJ012544G-00233): 10.10.0.2/24. IPv4 Forwarding ON. Config pushed.
XIQ
Port type SW_Mgmt_VLAN10 created. Access port, VLAN 10, Edge Port ON. For laptop direct connection to management network.
Issue
VLAN 10 SVI was DOWN — 0 ports. show iproute showed no U or f flag on 10.10.0.0/24. EXOS requires at least one port in link-up state for SVI to be active.
CLI Fix
Added port 1 tagged to VLAN 10. configure vlan SW_VLAN_10_Mgmt add ports 1 tagged on both switches. SVI came UP immediately — U + f flags confirmed in show iproute.
Quantum Fiber
Static route added: 10.10.0.0/255.255.255.0 → 192.168.0.28.
Blocker
Mac cannot ping 10.10.0.1. Quantum Fiber router does NOT route LAN→LAN traffic via static routes. Static routes only apply to WAN-bound traffic. Switch can ping Mac but Mac cannot reach VLAN 10.
⚠ Next Step — Direct Cable Required for VLAN 10 SSH

To SSH to 10.10.0.1 / 10.10.0.2, plug laptop directly into a switch port assigned SW_Mgmt_VLAN10 port type. Set laptop static IP to 10.10.0.10/24, gateway 10.10.0.1. Then SSH directly — no router involved.

Laptop static IP: 10.10.0.10 / 255.255.255.0
Gateway: 10.10.0.1
ssh admin@10.10.0.1   (SW1)
ssh admin@10.10.0.2   (SW2)

XIQ Steps

Step 1
XIQ → Network Policies → Test_SW1 → VLAN Attributes → Add VLAN 10, name SwMgmt
Step 2
Network Allocation → add 10.10.0.0/24 → SVI: SW1=10.10.0.1, SW2=10.10.0.2 (assign per device)
Step 3
Port Types → add VLAN 10 tagged to uplink port type (port 1) — NOT on AP_Port
Step 4
Push config via XIQ → both switches update simultaneously
Step 5
Quantum Fiber router → add static route: 10.10.0.0/24 → 192.168.0.28
Step 6
Test: SSH to 10.10.0.1 (SW1) and 10.10.0.2 (SW2) — confirm management reachable on VLAN 10
Critical constraints: VLAN name must be SwMgmt — "Mgmt" and "Management" are reserved keywords in EXOS and will fail silently or error. Do NOT remove VLAN 1 from uplink ports — XIQ loses connectivity. Test SSH to 10.10.0.1 BEFORE making any changes to VLAN 1.
