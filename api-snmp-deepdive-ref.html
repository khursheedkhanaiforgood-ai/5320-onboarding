Protocol overview

SNMP is an IETF-standardized Application-layer protocol for reading and writing management information on network devices. Three generations exist — each solving the limitations of the previous. Your 5320-16P (both Switch Engine and Fabric Engine) implements all three.

SNMPv1RFC 1157 · 1990
SNMPv2cRFC 1901 · 1996
SNMPv3RFC 3410–3418 · 2002

Why SNMP is no longer enough for configuration

SNMP was designed for polling and notification — it excels at read-only monitoring. Configuration via SNMP SET is fragile, lacks transactions, and has no rollback. The IETF responded with YANG + NETCONF + RESTCONF, which underpin modern network automation including ExtremeXOS RESTCONF on the 5320.

📐
YANG
RFC 7950

Data modeling language — defines structure and semantics of config + state data. Analogous to SMIv2 for SNMP MIBs, but richer (types, groupings, augmentations).

YANG 1.0RFC 6020
YANG 1.1RFC 7950 (current)
⚙️
NETCONF
RFC 6241

XML-RPC over SSH (port 830). Provides transactional config with candidate/running/startup datastores, locks, and confirmed-commit rollback.

TransportSSH (mandatory), TLS (optional)
EncodingXML
🔗
RESTCONF
RFC 8040

NETCONF semantics over HTTPS. GET/POST/PATCH/DELETE on YANG-modeled resources. EXOS implements this natively on the switch itself.

TransportHTTPS
EncodingJSON or XML

EP1 vs XIQ — management plane relationship

In the Horizon lab, EP1 is the SD-WAN/RDC component accessed via IPE on SW2 Port1, VLAN 4047. EP1 can operate in tandem with cloud XIQ or fully standalone. The APIs are complementary: XIQ API handles cloud-managed wireless and switch config; EP1 API handles on-prem policy enforcement, local SSID control, and site-specific ACLs.

XIQ API scopeCloud managed — APs, switches, network policies, global VIQ
EP1 API scopeOn-prem controller — local breakout, site policy, VLAN 4047 path
OverlapDevice onboarding, SSID config — but via separate endpoints and auth
⚠️ XIQ continuity note (May 2026): The Extreme community confirmed that all xAPI endpoints continue working unchanged through the XIQ UI → EP1 transition. EP1 is in active development with growing feature parity.

IETF RFCs — SNMP stack
RFC 1155
SMIv1 — Structure of Management Information
Defines how MIB objects are described. Foundation for all MIBs.
RFC 1157
SNMPv1 — A Simple Network Management Protocol
The original SNMP spec. Community string auth, UDP 161/162. Published May 1990.
RFC 1212
Concise MIB Definitions
Companion to SMIv1 — how to write MIB module definitions.
RFC 1213
MIB-II — Management Information Base for TCP/IP
The foundational MIB. Interfaces, IP, TCP, UDP, SNMP groups.
RFC 1901
Introduction to Community-based SNMPv2
The SNMPv2c framework — still community-string auth, adds GetBulk + Inform.
RFC 2578
SMIv2 — Structure of Management Information v2
Richer type system, 64-bit Counter64. STD 58.
RFC 2863
IF-MIB — The Interfaces Group MIB
Modern interface stats replacing MIB-II ifTable. Essential for port monitoring.
RFC 3410
Introduction to SNMPv3 Framework
High-level overview and applicability. STD 62.
RFC 3411
Architecture for SNMP Management Frameworks
Defines the modular SNMPv3 architecture — the subsystems model.
RFC 3412
Message Processing and Dispatching (SNMP)
How SNMP engines negotiate and dispatch messages across versions.
RFC 3413
SNMPv3 Applications
Defines command generators, command responders, notification originators.
RFC 3414
USM — User-based Security Model
Auth (MD5/SHA) and privacy (DES/AES) for SNMPv3. EXOS implements this fully.
RFC 3415
VACM — View-Based Access Control Model
MIB subtree access control. EXOS: configure snmpv3 add mib-view.
RFC 3416
Protocol Operations for SNMPv2
GetBulkRequest, InformRequest PDU definitions. Used in SNMPv2c and v3.
RFC 3418
MIB for SNMP
The SNMP agent's own MIB — stats on the SNMP engine itself.
RFC 3826
AES Cipher in SNMP USM
AES-128 privacy for SNMPv3. EXOS also implements 192/256 as proprietary extensions.
RFC 4133
ENTITY-MIB
Chassis, module, port, sensor hierarchy. Useful for 5320 hardware monitoring.
RFC 4188
BRIDGE-MIB
MAC address table, STP state per port.
RFC 4363
Q-BRIDGE-MIB
VLAN membership tables. Query which ports are in VLAN 20/30 on SW1.
IETF RFCs — NETCONF / YANG / RESTCONF
RFC 6020
YANG 1.0 — Data Modeling Language for NETCONF
First version of YANG. Defines module, container, leaf, list constructs.
RFC 6241
NETCONF — Network Configuration Protocol
XML-RPC over SSH. Transactional config with candidate datastore + commit.
RFC 7895
YANG Module Library
Allows clients to discover all YANG modules supported by a server.
RFC 7950
YANG 1.1 — Data Modeling Language
Current YANG spec. Actions, notifications, anydata. What EXOS RESTCONF uses.
RFC 8040
RESTCONF — HTTP-Based Protocol for YANG
NETCONF semantics over HTTPS. This is what ExtremeXOS Switch Engine implements natively.
RFC 8072
YANG Patch — Media Type for Datastore Patches
Atomic multi-edit patches in RESTCONF.
