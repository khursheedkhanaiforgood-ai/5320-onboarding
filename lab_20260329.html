Remaining Questions — Answer These Before Module B

Question 6 — Inter-Switch Trunk Port
You have 16 ports on each switch. Port 1 is the uplink to the modem. Ports 3/5 (SW1) and 13/15 (SW2) are for APs. Which port on each switch will be the trunk cable between Switch_1 and Switch_2 — and what VLANs should that trunk carry?
Think about it: you want VLAN 10 to be reachable on both switches for management. Should the data VLANs (20, 30, 50, 60) also cross that trunk, or only VLAN 10?
Question 7 — Two VLANs on One AP Port
You mentioned you could not add Corp1 and Guest1 simultaneously to a single port in the previous session. Before we get to execution — what is the difference between adding a port to a VLAN with the tagged keyword vs without it? And why does an AP need tagged frames to support two SSIDs on one cable?
Think about what happens when two VLANs both want to be the "native" VLAN on the same port. Can a port be untagged on two VLANs at the same time?
Question 8 — VLAN 10 Subnet & IPs
You've chosen 10.10.0.0/24 for management. What IP will you assign to Switch_1's VLAN 10 interface, and what IP for Switch_2? Will you assign them statically or via DHCP?
Management IPs should be static — you need to SSH to a predictable address even if the Default VLAN DHCP address changes.

AP note: Switch configuration is 100% CLI. AP SSID-to-VLAN mapping (Corp1→VLAN20, Guest1→VLAN30) must be done in XIQ — the AP3000 CLI does not support SSID configuration. XIQ steps are clearly marked below.

Step 1 — Pre-Reset: Decommission AP from XIQ

XIQ → Devices → select AP3000-A → Actions → Reset to Factory Default (or Decommission)
Why: if you skip this, XIQ re-pushes the old config the moment the AP comes back online.

Step 2 — Factory Reset SW1 (Console Cable)

# Connect console cable
ls /dev/cu.*
screen /dev/cu.usbserial-XXXXXX 115200

# Login (EXOS from previous session)
admin
[blank password]

# Wipe configuration (EXOS command — NOT "delete configuration primary")
unconfigure switch all
# prompt: Yes (capital Y)
Wait ~5 minutes. Switch boots into ZTP+ mode, gets DHCP IP from home router, checks in with XIQ.

Step 3 — Post-ZTP+: Find IP and SSH In

# Still on console after reboot — login
admin
[blank password]

# Complete setup wizard
# Key answers: N N N N Y Y N Y  (same as original onboarding)

# Find the DHCP IP assigned by home router
show ipconfig

# Note 192.168.0.x — this is your SSH target AND your Quantum Fiber static route next-hop

# Open new terminal — SSH in
ssh -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa admin@192.168.0.x

Step 4 — SW1: Create VLANs and Configure Ports

# Create VLANs
create vlan Corp1 tag 20
create vlan Guest1 tag 30

# Add AP port — BOTH tagged so Port 3 carries both VLANs simultaneously
configure vlan Corp1 add ports 3 tagged
configure vlan Guest1 add ports 3 tagged

# Assign IP interfaces
configure vlan Corp1 ipaddress 10.20.0.1/24
configure vlan Guest1 ipaddress 10.30.0.1/24

# Enable PoE for AP
enable inline-power ports 3

Step 5 — SW1: Enable IP Forwarding (All Routed VLANs)

# Required on ALL three — lesson from March 27
enable ipforwarding vlan Default
enable ipforwarding vlan Corp1
enable ipforwarding vlan Guest1
Default VLAN must be enabled — return traffic from internet arrives here before being routed to Corp1/Guest1.

Step 6 — SW1: Configure DHCP for Both VLANs

# Corp1 DHCP
configure vlan Corp1 dhcp-address-range 10.20.0.100 - 10.20.0.200
configure vlan Corp1 dhcp-options default-gateway 10.20.0.1
configure vlan Corp1 dhcp-options dns-server primary 8.8.8.8
enable dhcp ports 3 vlan Corp1

# Guest1 DHCP
configure vlan Guest1 dhcp-address-range 10.30.0.100 - 10.30.0.200
configure vlan Guest1 dhcp-options default-gateway 10.30.0.1
configure vlan Guest1 dhcp-options dns-server primary 8.8.8.8
enable dhcp ports 3 vlan Guest1

Step 7 — SW1: Save Configuration

save configuration
# prompt: y

Step 8 — Factory Reset AP3000-A (Physical Button)

1. Plug AP3000-A into SW1 Port 3 (PoE now active)
2. Hold reset button 10–15 seconds until LED flashes rapidly
3. AP reboots → gets 192.168.0.x from home router via Default VLAN (untagged on Port 3)
4. AP reaches XIQ and appears as "New Device" for adoption
Wait 2–3 minutes for AP to fully boot and check in to XIQ.

Step 9 — XIQ: SSID Mapping (Cannot Be Done via CLI)

1. XIQ → Configure → Network Policies → New Policy → name: SW1-AP-Policy
2. Add SSID: Corp1 → VLAN 20 → WPA2/WPA3 passphrase
3. Add SSID: Guest1 → VLAN 30 → WPA2/WPA3 passphrase
4. Assign AP3000-A to SW1-AP-Policy
5. Push / Deploy → wait for AP to apply config (~1 min)
AP will now broadcast both SSIDs. VLAN tags are sent from AP to SW1 Port 3.

Step 10 — Quantum Fiber: Add Static Routes for SW1

Quantum Fiber UI → Advanced → Static Routing → Add:
10.20.0.0 / 255.255.255.0 → 192.168.0.x (SW1's IP from show ipconfig) 10.30.0.0 / 255.255.255.0 → 192.168.0.x (same SW1 IP)

ipforwarding on Default VLAN — mandatory after every factory reset

Return traffic from internet arrives on VLAN 1 (Default). Without ipforwarding enabled here, clients get DHCP + gateway but Safari times out. Not set by XIQ. Must be done via CLI.

enable ipforwarding vlan Default
save configuration
Default route is automatic via DHCP — do NOT manually add it

EXOS installs a BOOTP default route automatically when the switch gets its management IP via DHCP. Check with show iproute — look for origin bo. Only enable ipforwarding vlan Default is manual.

CommandCorrect Syntax
DHCP address rangeconfigure vlan <name> dhcp-address-range 10.20.0.100 - 10.20.0.200 — dash separator with spaces required
DHCP gatewayconfigure vlan <name> dhcp-options default-gateway 10.20.0.1 — NOT "gateway", NOT "router"
DHCP DNSconfigure vlan <name> dhcp-options dns-server primary 8.8.8.8
Enable DHCPenable dhcp ports 3 vlan <name> — port number required, not optional
IP forwardingenable ipforwarding vlan Default — run on every VLAN that needs routing, incl. Default
XIQ vs CLI boundary (confirmed): XIQ pushes VLAN creation, SVI IPs, IPv4 Forwarding, port types, AP SSID policies. CLI required for: DHCP server, default-gateway DHCP option, enable ipforwarding vlan Default, DNS. Never add VLAN 1 to XIQ Routing section.
